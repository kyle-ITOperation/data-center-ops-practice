# 异常处理案例（Incident Handling）

## 案例1：Nginx服务无法访问

【问题现象】
- curl访问失败  

【排查步骤】
1. 检查服务状态（systemctl）  
2. 查看日志（tail error.log）  
3. 确认服务未运行  

【处理过程】
- 执行服务重启  

【处理结果】
- 服务恢复正常  

【总结】
通过基础检查可以快速定位问题，并恢复服务

## 案例2：服务器无法ping通

【问题现象】
- ping请求失败  

【排查步骤】
1. 检查服务状态（正常）  
2. 检查网络连接情况  
3. 判断为网络异常  

【处理过程】
- 未进行操作  

【处理结果】
- 记录问题并上报  

【总结】
对于非权限范围内的问题，应及时上报处理
